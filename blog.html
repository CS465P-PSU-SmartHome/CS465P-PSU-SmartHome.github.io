The final project assignment for undergraduate students is to create a front-end web app utilizing web application APIs such as Google Map API, Open Weather API, Spotify API, etc. My first impression was to create a home dashboard that could be customized to display or control personalized application in a web page. This inspiration was coming from my recent purchase of the Nest thermostat. I wanted to have my room temperature displayed on the web and control it utilizing Nest API, but sadly Nest API is eol and it’s in the transit to Google Assistant API so I decided not to use Nest API.

I started the project with some basic html tags and designed the navigation bar using bootstrap. It was pretty straight forward since we have learned bootstrap in class. The first API I tried was the openweather API. I first registered an account at OpenWeatherMap.org and generated an API key. The API usages on the webpage is well documented and I hardcoded Portland location for the API request since I wanted to use this web app at home only. Converting the API response back from the request to a json format variable made writing javascript easier. By assigning IDs to each of the elements such as weather, temperature, wind speed, humidity, we can modify the innerHTML using document.getElementByID.

Then, I added an embedded twitter time-line widget by visiting publish.twitter.com. The code is easily generated and modified. I then added a Google Map on my dashboard web app to display the time it would take me to get to school from home, and showing possible routes to save time. Using Google API is straight forward however I was running into issues where I had to link to a credit card in order to use the API. otherwise it would be ‘development only’ water marks on the map. Later I added two input boxes for users to check out the travel time between different locations, here I encountered issues with accessing the element with getElementById. By debugging it with the console on Google Chrome Browser I noticed that the click event of the button was not able to access the iframe element with getElementByID because it was always returning null object. After googling and reading the comments, I solved it by putting the addEventListener script section at the end of the tag because we needed to initialize the element first before we can access it.

The most time consuming part of this project was the styling. Utilizing css to modify the location and the style of each widget was not fun. I wanted to create a nice looking web application but some of the widgets’ padding were off by like 1px, and the block alignments were giving a headache.

Exploring the Google Cloud Platform, I noticed that there were a lot of things we can do with just the Google Maps API. The distance matrix API call would return the distance between the origin and destination, as well as the time it takes to get there. The direction API provides detailed turn-by-turn navigation that could help developers to implement navigation map.

Finally, I added the outlook calendar using an iframe. With the calendar displayed on the web, I could now see my daily schedules all in one dashboard making life easier. The outlook calendar also syncs up with my outlook account automatically so whenever I make a change to my outlook calendar then it would be updated in the web page as well.

The journey of the final project has been fun and I believe the home dashboard web application has its potential to expand. My next step would be to embed Spotify player and news feed onto the webpage, then redesign or use some templates for the dashboard.
